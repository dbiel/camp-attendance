**Reconciled conflict — all-resolved combined link:** sec-external (critical_gap) says **uniform 404**; sec-authz (suggestion) says 200 empty array. Critical_gap outranks suggestion, and the binding principle (a forwardable URL exposing a flagged **minor** should die the moment it's no longer needed) wins. **Decision: all-resolved → uniform 404.** The viewer copy softens to "This link is no longer active — the report may have been resolved, or the link expired." No enumeration signal; link auto-dies.

**Reconciled conflict — Anthropic roster egress:** the draft proposed pre-filtering the roster. **Reversed.** database + workflow show pre-filtering (a) breaks the `cache_control` ephemeral prefix (can *raise* cost), (b) drops exactly the misspelled/nickname candidates fuzzy-match exists to catch, and (c) is impossible for B1 multi-person (candidates unknown until parse). **Keep the full cached roster.** It already excludes student phones and dorm *room* (only `dorm_building`). The only minimization adopted: trim parent phone numbers from the contacts block to name+role unless a phone-match is needed (low-risk, cache-safe). This is a *suggestion*, not a critical_gap, so it's deferred to Phase 3 as a nicety.
